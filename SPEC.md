# Groq哥哥 LINE Bot — Technical Specification

**Version:** 2026-03-15  
**Stack:** FastAPI + Groq API + LINE Messaging API v3 + SQLite  
**Deploy:** Railway (persistent volume at `/data/chat_history.db`)  
**Timezone:** Asia/Taipei (UTC+8)

---

## 1. Architecture Overview

```
LINE User
  │
  ▼
LINE Webhook → FastAPI /webhook
  │
  ▼
process_event()
  ├── AUDIO  → Whisper STT → clean_transcript(gpt120b) → reminder/LLM
  ├── IMAGE  → Scout vision → save context → reply
  └── TEXT   → handle_command() OR reminder_nlp OR resolve_model → LLM
  │
  ▼
reply_message() → split chunks if > 4990 chars → LINE (max 5 chunks)
```

## 4. Auto-Router Logic

```python
resolve_model(user_id, user_text):
  1. User manually set model → use that (skip auto-route)
  2. Text > 500 chars, no "?" → llama70b (summarization)
  3. _needs_realtime(text) → compound-mini
  4. classify_query(text) via llama8b → ROUTE_MAP
```

**ROUTE_MAP:**
| Category | Model | Trigger |
|---|---|---|
| simple | llama8b | greetings, yes/no, short facts |
| creative | llama70b | writing, translation, summarization |
| reasoning | qwen | math, logic, code, analysis |
| hard | gpt120b | ambiguous, multi-domain, deep thinking |
| search | compound-mini | real-time data only |

**Realtime keywords** (→ compound-mini bypass):
`hôm nay, bây giờ, hiện tại, mới nhất, today, now, latest, giá, price, tỷ giá, thời tiết, weather, tin tức, news, 今天, 現在, 最新, 價格, 新聞, 天氣`

## 5. Database Schema

```sql
-- Conversation history
history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id TEXT,
  role TEXT,       -- "user" | "assistant"
  content TEXT
)

-- Per-user model preference
user_settings (
  user_id TEXT PRIMARY KEY,
  model_key TEXT NOT NULL
)

-- Per-user max_tokens setting
user_settings2 (
  user_id TEXT PRIMARY KEY,
  max_tokens INTEGER NOT NULL DEFAULT 800
)

-- Conversation summary (auto-generated every 20 messages)
summary (
  user_id TEXT PRIMARY KEY,
  content TEXT,
  updated_at INTEGER  -- unix timestamp
)

-- Reminders
reminders (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id TEXT NOT NULL,
  message TEXT NOT NULL,
  fire_at INTEGER NOT NULL,   -- unix timestamp UTC+8
  repeat TEXT DEFAULT NULL,   -- null | "daily" | "weekly" | "monthly"
  done INTEGER DEFAULT 0      -- 0=pending, 1=done/cancelled
)

-- User profile (injected into system prompt)
user_profile (
  user_id TEXT PRIMARY KEY,
  name TEXT,
  occupation TEXT,
  learning TEXT,
  notes TEXT
)
```

## 6. Memory System

**get_history_with_summary(user_id):**
- Returns: `[summary_user_msg, summary_assistant_msg, ...5 recent messages]`
- Summary auto-generated by `llama8b` every `SUMMARY_TRIGGER=20` messages
- Max 4000 chars of recent history

**build_system_prompt(user_id, model_key):**
- Base: `get_system_prompt(model_key)` from `prompts.py`
- Appends user profile if exists: name, occupation, learning, notes
- Per-model suffixes: REASONING_SUFFIX, CREATIVE_SUFFIX, SEARCH_SUFFIX

## 7. Audio Pipeline

```
AudioMessageContent
  → call_groq_whisper() [whisper-large-v3-turbo]
  → clean_transcript(gpt120b) [fix Whisper errors, temperature=0.0, max_tokens=300]
  → check _REPLY_TRIGGERS ("hello", "不好意思")
      YES (wants_reply=True):
        → strip trigger prefix + leading punctuation
        → parse_reminder_nlp() → if reminder → reply
        → else → resolve_model() → call_groq_text() → reply
      NO (transcribe-only):
        → parse_reminder_nlp() → if reminder → reply
        → else → save "[Voice]: {transcript}" → reply transcript only
```

**_REPLY_TRIGGERS:** `("hello", "不好意思")`

## 8. Reminder System

**Natural language (parse_reminder_nlp):**
- Uses `llama8b`, temperature=0.0, max_tokens=100
- Returns JSON: `{is_reminder, message, time (HH:MM|null), date (DD/MM/YYYY), repeat}`
- Python calculates `fire_at` timestamp (NOT llama8b) to avoid timezone errors
- If `time=null` → asks user for time
- Detects implicit reminders: "tôi có họp lúc 14h" → is_reminder=true
- Converts: "7h tối" = 19:00, "7h sáng" = 07:00

**Slash command (/remind):**
- `/remind HH:MM [daily|weekly|monthly] content`
- `/remind HH:MM [tối|chiều|sáng] [ngày DD/MM[/YYYY]] content`
- PM detection: checks full arg string for tối/chiều/pm → +12h if hour < 12
- Date detection: regex `(\d{1,2})[/-](\d{1,2})` in rest string
- `/remind list` → show pending
- `/remind <id> cancel` → cancel

**reminder_loop():** Background task, checks every 30 seconds, pushes LINE notification, updates `fire_at` for repeating reminders.

**_next_fire(fire_at, repeat):**
- daily → +1 day
- weekly → +7 days
- monthly → +1 month (manual calculation, no dateutil dependency)

```
user_max = get_user_max_tokens(user_id)  # from user_settings2

    max_tok = user_max  # user override via /long
else:
    max_tok = 1500 if model_key in (qwen, gpt120b, gpt20b) else 800

**reasoning_effort:** `low` for gpt120b (saves tokens)

## 11. Reply Handling

```python
# Split long reply into multiple LINE messages
chunks = []
while len(reply) > 4990:
    cut = reply.rfind(' ', 0, 4990)  # cut at nearest space
    chunks.append(reply[:cut])
    reply = reply[cut:].strip()
chunks.append(reply)
chunks = chunks[:5]  # LINE max 5 messages per reply
```

## 12. strip_markdown()

Removes for LINE output (no markdown support):
- `<think>...</think>` tags (reasoning models)
- Unclosed `<think>` tags fallback
- Fenced code blocks ` ``` `
- Inline code `` ` ``
- Headings `#`
- Bold/italic `**` `*` `__` `_`
- Strikethrough `~~`
- Bullet lists `- ` `* ` → `• `
- Horizontal rules `---`
- Links `[text](url)` → `text (url)`
- Blockquotes `>`
- Collapse 3+ blank lines → 2

## 13. System Prompt (prompts.py)

**Base SYSTEM_PROMPT:** 繁體中文 teacher + career mentor persona "Groq哥哥" from Taiwan.

**Rules in prompt:**
- Language matching (Vietnamese/Traditional Chinese/Japanese)
- No Markdown (LINE chat)
- Complete answers, no truncation for brevity
- Follow-up A/B/C options after each answer

**Suffixes by model_key:**
- `qwen, gpt120b, gpt20b` → `REASONING_SUFFIX` (output final answer only, hide thinking)

## 15. Known Issues / Gotchas

1. **Compound models:** Require last history message role = "user" → trim assistant tail
2. **Compound model IDs:** Must be `groq/compound` and `groq/compound-mini` (not `compound-beta`)
3. **Duplicate functions:** `get_user_max_tokens`, `set_user_max_tokens`, `get_user_profile`, `save_user_profile` appeared twice — keep second occurrence only
4. **Timezone:** All datetime must use `datetime.now(TZ)` and `datetime.fromtimestamp(x, tz=TZ)`
5. **Reminder timestamp:** llama8b calculates wrong timestamps → Python calculates fire_at from llama8b's HH:MM + DD/MM/YYYY output
6. **Voice transcript cut:** Whisper sometimes truncates last word → clean_transcript may not recover
7. **_needs_realtime false positive:** Keywords like "hôm nay" in reminder context → check reminder BEFORE realtime routing
8. **user_settings2 table:** Separate from user_settings to avoid migration issues
